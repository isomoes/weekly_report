# Weekly Report Guide

Use this guide for weekly reports, progress reports, and technical update writing.

## Goal

Write clear academic or technical progress paragraphs that document work performed, methods used, challenges addressed, and measurable outcomes.

## Style

- Use clear, concise academic language.
- Maintain a professional, technical tone.
- Prefer paragraph form instead of bullet points or numbered lists unless the user requests otherwise.
- Convert enumerated notes into flowing narrative paragraphs with smooth transitions.
- Use passive voice only where it improves academic fit; otherwise prefer the clearest phrasing.

## Objectivity

- Never use self-evaluative terms such as `significantly`, `greatly`, `remarkably`, `excellent`, or `outstanding` unless they come from quoted source text that must be preserved.
- Do not praise the work. State what was done.
- Replace subjective assessments with objective facts, measurements, or scope.

## Technical Detail

- Name specific methods, algorithms, tools, modules, functions, or systems when available.
- Include quantitative details such as counts, timings, benchmarks, or scale.
- Describe technical challenges and how they were addressed.
- State scope clearly, such as number of components changed or comparisons performed.

## Highlighting Rules

If the target environment supports the author's existing markup, use highlighting sparingly:

- Use `#redt` only for the most critical result, issue, or breakthrough.
- Use `#bluet` for secondary important technical details.
- Do not over-highlight. Most text should remain untagged.

If the user is not writing for that markup environment, keep the emphasis as plain prose instead of forcing the tags.

## Paragraph Quality Bar

Aim for paragraphs that:

- have a clear topic sentence
- stay logically ordered
- connect actions to outcomes
- remain concise while still technically specific

## Good Direction

Prefer this kind of sentence:

"Benchmarked three batching strategies for the training pipeline and selected the configuration that reduced epoch time from 92 min to 61 min after the CUDA memory leak was removed."

Avoid this kind of sentence:

"Greatly improved the training system and achieved significantly better performance."
